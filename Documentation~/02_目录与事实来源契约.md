# HoNpr 目录与事实来源契约

本文定义 package 目录职责。目录边界是工程约束，不是建议。

## 顶层目录

| 路径 | 职责 | 禁止事项 |
| --- | --- | --- |
| `Documentation~/` | 系统设计契约、边界说明、迁移决策、验收清单。 | 放派生表、生成器临时输出、实现文件长注释备忘。 |
| `ShaderSystem/` | Shader 生成系统的声明层和派生表。 | 放大段 HLSL 主体、Unity 材质实例状态。 |
| `Shaders/ShaderLibrary/` | 手写 HLSL 实现、ABI wrapper、lobe、composite、lighting helper。 | 放 generated shader，读取 Inspector 状态。 |
| `Shaders/Generated/` | 最终生成的 `.shader` 审查产物。 | 放隐藏中间 HLSL 大包、手写长期源文件。 |
| `Shaders/HoToon/` | 旧 HoToon 或兼容参考 shader。 | 作为 HoNpr 新 shader ABI 的来源。 |
| `Editor/` | 生成器、校验器、材质 UI、导入设置。 | 硬编码 preset 结构事实。 |
| `Textures/` | package 自带纹理资源。 | 放声明文件或生成代码。 |

| 路径 | 文件类型 | 事实来源级别 | 职责 |
| --- | --- | --- | --- |
| `Contract/` | `.md` | 上游索引 | 索引 HoRP 语义、resource、pass、ABI include。 |
| `Includes/` | `.honprinclude` | 人工源 | include 别名注册，避免 block 直接写路径。 |
| `Features/` | 文件夹、`.honprblock`, `.honprparams`, `.honprui`, `.md` | 人工源 | 以文件夹作为 feature identity，集中放组分身份、参数、默认值、UI 归属和实现入口；preset UI profile 放在 `Features/PresetUi/`。 |
| `Templates/` | `.honprtemplate`, `.shader.template` | 人工源 | ShaderLab、pass、render state、pragma、entry slot 骨架。 |

## Feature Ownership 追踪规则

`ShaderSystem/Features/<Domain>/<FeatureId>/` 本身就是 feature 的身份。目录存在即表示这个 feature 是一个可维护对象，不需要再用独立 manifest 文件证明身份。

一个 feature 的完整信息包括：

- FeatureBlock：身份、输入、输出、依赖、capability、producer / consumer、legacy source。
- ShaderLibrary / Assembly：实现入口和稳定调用入口。
- Preset：feature 属于哪些 domain / archetype / tier，以及参与哪些 pass。
- MaterialUi：参数白名单、UI 分组、默认展示规则。preset 级 UI profile 位于 `Features/PresetUi/`。
- Contract：上游 HoRP semantic、resource、pass 或 ABI 索引。
- Generator：校验规则、source mapping 和生成结果。

## Material UI renderState 写法

`*.honprui` 的 `renderState` 是只读展示声明，不是材质实例参数，也不能用普通 `property` 暴露 queue、blend、stencil、zwrite、ztest、cull 或 lightmode。多 pass / 多阶段 shader 仍写成一条 `renderState` statement，但每个字段的字符串必须使用稳定分隔符，方便 Inspector 做多行优化显示：

- `; ` 分隔 pass 或阶段，例如 `depth "Outline ZWrite On; Forward/Depth/Shadow ZWrite On; ZTest LEqual"`。
- ` / ` 分隔同一阶段内的并列 pass 集合，例如 `Forward/Depth/Shadow ZWrite On`。
- `, ` 分隔同一 pass 或阶段内的多个 state 子项，例如 `ZWrite On, ZTest LEqual`。

约定顺序是先写 pass / 阶段，再写 state。字段含义必须来自 preset / template 的静态结构；如果实际 shader 增删 pass 或 render state，先改 preset / template，再同步 `renderState` 文本，不能只改 UI 文本。

推荐形态：

```text
ShaderSystem/Features/Stylized/RimLightLilToon/
  README.md
  Block.honprblock
  Parameters.honprparams
  Defaults.honprdefaults
  Ui.honprui
```

生成器只读取 `ShaderSystem/Features/**/*.honprblock`。同一个 `MaterialBlock.*` 只能存在一份声明。`ShaderSystem/FeatureBlocks/` 已废弃，不再保留目录。

禁止依靠全文搜索、命名习惯、RT 槽位、keyword 或 UI 折叠项来补全 feature 语义。跨文件追踪必须靠稳定 ID、文件夹边界和显式引用。

## 派生表

这些文件由源声明生成：

- `ShaderSystem/Templates/TEMPLATE_TABLE.md`
- `ShaderSystem/Features/FEATURE_BLOCK_TABLE.md`
- `ShaderSystem/Presets/PRESET_TABLE.md`
- `ShaderSystem/MaterialUi/MATERIAL_UI_TABLE.md`

规则：

1. 派生表不手改。
2. 派生表不参与结构决策。
3. 派生表用于 code review、文档浏览和 CI diff。
4. 表格内容错误时，修改源声明或生成器，不修改表格。

## Generated 目录

`Shaders/Generated/` 只放 Unity 可以直接编译的最终 `.shader`。

允许：

- source mapping 头。
- `Shader` / `Properties` / `SubShader` / `Pass`。
- pass tags 和 render state。
- 少量固定 pragma。
- include 稳定 ShaderLibrary 或 Assembly 入口。
- 绑定稳定 vertex / fragment entry。

禁止：

- 生成 `Generated/*.hlsl` 中间大包。
- 手写长期维护代码。
- 内联完整 lighting / lobe / composite 主体。
- 大量结构宏和 `#if defined(HONPR_HAS_...)`。
- 旧 `_lil*`、`_HoAov*`、`HoAOV`、`lilToonOIT` 作为新 ABI。

## ShaderLibrary 目录

ShaderLibrary 是复杂度的主要容器。

建议结构：

```text
Shaders/ShaderLibrary/
  HoNprCommon.hlsl
  Lighting/
  StandardSurface/
  StylizedSurface/
  SemanticSurface/
  Transparency/
  Composite/
  Debug/
  Assemblies/
```

`Assemblies/` 是预编译组合层，负责把多个底层实现收敛为 preset 级稳定入口。当前如果尚未完全建立 Assembly 层，`CharacterToonLilToonSourceInline.hlsl.template` 这类带来源身份的大模板只能视为迁移过渡，不应成为长期结构。

Assembly 应输出类似：

```hlsl
HoNprCharacterToonVaryings HoNprCharacterToonVert(...);
half4 HoNprCharacterToonStandardForward(...);
half4 HoNprCharacterToonStandardOutline(...);
HoNprAovOutput HoNprCharacterToonStandardAov(...);
HoNprOitOutput HoNprCharacterToonTransparentOit(...);
```

## Editor 目录

Editor 可以：

- 读取声明。
- 建立 token。
- 校验引用。
- 生成派生表。
- 生成 `.shader`。
- 刷新 Unity asset。
- 渲染材质 UI。

Editor 不能：

- 按 preset 名称硬编码结构。
- 从材质 Inspector 当前值推导 pass / block / shader 文件路径。
- 靠 C# 字符串拼接完整 HLSL 主体。
- 绕过统一写文件入口。

所有生成输出必须统一行尾。当前约定为 LF。

## 文件命名

规则：

- 设计文档使用编号前缀：`00_`、`01_`。
- DSL 源文件使用稳定英文 ID，避免路径编码问题。
- 用户可见 shader 名称使用 HoNpr 语义：`HoNpr/Character/Toon_Standard`。
- 来源身份保留在 block 名称中：`ToonDiffuseRampLilToon`、`MatCapLilToon`。
- 迁移原型必须显式带 `Source`、`Legacy` 或 `Prototype` 语义，不能伪装成默认入口。
